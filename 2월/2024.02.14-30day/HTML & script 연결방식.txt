* HTML & script 연결방식
- 인라인 스크립트
- 내부 스크립트
   닫는 body 태그 바로 위에 script 파일을 적용하는방법
- 외부스크립트
    > 웹브라우저가 개발자가 작성한 html문서를 읽는다 
       = parsing 한다라고한다 (parse = 분석)
    > 웹브라우저가 script 문서를 마주한 순간 html 문서의 parsing 일시정지 후 script 빠저들어간다 = fetching = 웹브라우저가 우리가 작성한 script파일을 가져가서 읽는다 
    > 웹 브라우저가 script 문서를 다 fectiong 하면
       => ezecuting (실행)
     
     > body 안에 내용물을 가져와야하는 순간에는
         <script defer src="./script.js"></script>
       body 안에 내용물이 없어서 안 가져와도 되는 순간에는
          <script src="./script.js"></script>

     > JS 기본적으로 동기처리 방식 실행 => 스크립트 언어 => 1차선언어

 - defer - 가장 강력하다 - 비동기 작업을한다 
   패칭하면서 js도 같이 파칭한다
   스크립트 의존도가 높은 파일일수록 필요하다
   파씽이 끝날때까지 기다려줬다가 같이 출력해준다
 - async - 얘도 비동기처리한다
   스크립트 먼저끝나면 
   스크립트가 여러개있을때 먼저읽는애들 먼저 출력할때  
   제이쿼리 슬릭슬라이더 패더라이더 페이지스크롤 이팩트..
   다수의 스크립트가 각각이 연관이 없고 
   읽는순서대로 먼저 출력해야될때 사용한다
 - 아무것도 안쓸때
   body 안에 아무것도 없을때 
   선택자를 불러올게 없을때
   defer를쓰게되면 파씽이 끝나도 얘는 기다리느라 화면에 출력이 안된다
 - 이벤트 써야될때

<!DOCTYPE html>
<html lang="ko">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <link rel="stylesheet" href="./style.css" />
    <title>Document</title>
  </head>
  <body>
    
    <script src="./script.js"></script>
  </body>
</html>

3.CSS3
 1) 색상
    - 표현단위 : HEX코드(표현빈도가 가장높다) / 컬러네임 / rgb / rgba / hsl / hsla
       (HEX코드는 레드를 #f00 블루는 #00f 그린은 #0f0)
    - 인간은 숫자를 셀때, 1~10 /11~20 => 10진법
    - 컴퓨터는 16진법 => 총 16개의 숫자 단위를 활용해서 표현
      컴퓨터는 0부터 시작이다 (1월이 0, 12월이 11..)
      1~16 / 0,1, .. 9, a, b, c, d, e, f (미니멈0, 멕시멈f) 
    - 바이너리코드 (o냐 x냐 / 0이냐 1이냐)
    
    - 컬러의 삼원색 => 3가지 컬러의 조합으로 완성된다
    - "R"ed / "G"reen /"B"lue =>RGB

    - #RRGGBB => 1개의 최종 컬러를 도출 
    - #ff0000 -> #f00
      (레드컬러가 최대치 ->f / 그린과 블루는 넣고싶지않아 ->0000
        똑같으면 하나만 입력하게 하자 #FF0000->#f00)

    - RGB
      background: rgb(211, 211, 211); 
      컴퓨터가 조합할수 있는 수 최대가 255이다
      #ff0000 255 - 16*16 ?
      background: rgb(255, 255, 255); => 색 다 넣어서 흰색이 된다
      background: rgb(0, 0, 0); => 색 다 빼서 검정이 된다
      그래서 흰색은 #FFF 검정은 #000

    - rgba : rgb에 a알파값
      a는 투명도 0이 최저 1이 최대 0.5는 절반
      background: rgba(0, 0, 0, 0.5);

    - hsl : hue(색상)   /   saturation(채도) / lightness(명도)
            0~255까지      높다 낮다           밝기를 올렸다 
    - hsla : hsla에 a알파값


 2) url() : 함수 ( ()가 있다 ->함수다)
     오락실은 오락실 게임기에 동전을 넣어야 작동한다
     함수는 ()안에 어떤 값을 넣어야 작동한다
     - 함수 안에는 매개변수 or 인수 or 인자값 이 들어가야한다
     - 경로라는 매개변수 or 인수 or 인자값 이 들어가야한다
     - url( "문자열로 되어진 경로" )
      
      ( css에서 background 속성값 나열 순서가 중요하다
       순서헷갈리면 mdn가라 
       https://developer.mozilla.org/ko/docs/Web/CSS/background 명세
       https://drafts.csswg.org/css-backgrounds/#the-background )
    

 3) 가시속성 : 가시성 (*눈에 보여질 수 있는 속성) 
    hml태그 : 2가지 (block태그 / inline태그)
  인라인은 하나의 행에 나눠쓰겠다
  블락 하나의 행 다 쓰겠다 
 
 > inline vs inlie-block :
  * inline :
   - width & height 값을 적용받지 않는다
   - margin값의 경우에 좌, 우측 값만 적용받는다
  * inline-block : (인라인말고 얘를 더 쓴다)
   - width & height 값을 모두 적용받는다
   - margin 값의 경우에 상,하,좌,우측 모든 값을 적용받는다 
 * dislplay vs visibility :
   - diplay : 영역의 흔적 자체를 없애거나 생성
   - visibility : 영역의 흔적은 살려두면서 보여지지 않게 할 수 있다.
 * opacity : 태그이 투명도를 조정할 수 있다.
   (0~1까지 줄 수있다 => 0-투명상태 / 1-100%불투명상태)
 

 4) 박스 속성 :
    - 박스 안에 컨텐츠 와 박스와의 관계를 잘 이해!!
    - margin : border를 기준으로 바깥쪽 영역을 의미한다
    - padding : border와 contents 사이의 영역을 의미한다
  *margin과  padding은 기본적으로 상,하,좌,우 값을 가져가는데 이때, 각 위치별    값을 상이하게 적용할 수 있다.
  *margin & padding :
    1개일때 : 공통
    2개일때 : 상하.좌우
    3개일째 : 상,좌우,하
    4개일때 : 상,우,하,좌 (상에서부터 시계방향)
   *box-sizing :
    - content-box : 안쪽에 정의해놓은 컨텐츠의 사이즈는 절대로 건드리지 않은 상태에서 마진, 패딩값을 추가하는 개념 (콘텐츠 사이즈에서 추가)
    - border-box : 안쪽에 정의해놓은 컨텐츠의 사이즈가 바깥쪽 보더를 기준으로 마진과 패딩값에 따라서 가변적으로 변경되는 개념

 8) Position(*위치) 속성
   - 절대 위치 좌표 : html에 여러개 복수 태그가 삽입 되었을 때, 먼저 입력한 태그가 먼저 출력 => 이걸 무시하고 => 절대 위치값을 부여하고 => 적용된다
   - 상대 위치 좌표 : 상대 html 태그가 입력된 위치를 기준으로 내 위치값을 지정하겠다는 의미
   - CSS3 Animation : 2D그래픽이 아니라 3D그래픽이다
     2D와 3D의 차이는 공간감의 차이다
     2D -x,y축  3D-x,y,z축  
     모니터가까워지면 가까워질수록 z축이 커진다 멀면 z축 작아진다

  자식요소에 position: absolute (*절대위치좌표)를 적용한 상태에서 해당 자식요소의 부모에게 position: relative (*상대위치좌표)를 적용하게 되면, 자식요소는 부모요소를 기준으로 절대위치좌표를 갖게된다

만약, html을 통해서 작성한 도형에 별도의 position값을 적용하지 않는다면, 기본 default값은 어떤 position 속성이 적용되는 것일까?
 default값은 position: static으로 
 그래서 position: static은 안 쓴다 (기본값하려면 아무것도 입력 안하면되니까)

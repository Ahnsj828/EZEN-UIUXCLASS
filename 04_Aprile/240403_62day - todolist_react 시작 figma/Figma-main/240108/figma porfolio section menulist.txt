*피그마를 활용한 포트폴리오 순서
(무 -> 유 창조 크리에티브 제작물)
(실제로 기능구현, 앱 런칭 가정)
(어떤 목적 / 목표 / 기능 구현 서비스)

*포트폴리오 제작 목적
- 피그마를 통해서 서비스 기능 구현 이미지 & 텍스트 효율적으로 표현 Skill

*Contents

1) Intro Section
- 내가 구현하고자하는 서비스의 간략한 소개 (*서비스에 대한 상징적인 섹션)
ex. 벗 앱 서비스 기획.런칭

2) Background Section
- 해당 기능이 왜 탄생하게 되었는가를 소개.설명하는 영역
- 설문조사 및 리서치 통계 수치 데이터
- %, 설문조사 응답 패널 숫자
ex. 온라인을 통한 자유로운 만남 트랜드
(MZ 세대의 커뮤니티 모임방식 추세)

3) User Research Section
- 구현하고자 하는 서비스와 유사한 기존서비스와의 차별화 포인트를 도출하기 위한 토대마련
ex. 우리가 출시하고자 하는 서비스와 유사한 기존 서비스가 있다라고 한다면, 차별화가 필요한데, 실제로 이 서비스를 사용할 기존 유저를 대상으로 불편한점과 추가되었으면 하는 기능들을 설문조사 형태로 수집

ex. 현재 내가 있는 위치 반경으로 이웃을 찾을 수 있는 기능이 기존에 없어서 불편
ex. 프로필 사진 영역이 매우 작아서 불편
ex. 나의 관심사 영역과 비슷한 친구를 찾을 수 있는 기능, 기존 서비스에 없었음

4) Persona Section
- persona : 연극 사용했던 단어. 우리의 서비스를 사용할 것으로 예상되는 User를 보다 구체화해서 이들이 이 서비스를 사용했을 때, 어떤 만족을 할 수 있을것인가를 시뮬레이션 형태로 보여주는 영역

ex. 페르소나1 = 21세. 남자 대학생
(*취업하고자 하는 기업의 관심사가 비슷한 친구가 있다면 만나서 스터디를 하고 싶었는데, 너무 좋은 서비스 같아요!)

=> 취준생들에게도 좋은 서비스

ex. 페르소나2 = 27세. 여자 직장인
(*퇴근 후 가볍게 소통할 수 있는 동성친구 찾기가 참 힘들었는데, 위치기반 서비스가 되어서 친구찾기 참 쉬워요~!!)

=> 오피스 우먼들에게도 좋은 서비스

ex. 페르소나3 = 32세. 남자 직장인
(*평소 헬스장을 다니는 편인데, 서로 운동 코칭해줄 수 있는 친구가 필요했는데, 정말 좋아요)

=> 직장 외 친구사귀기에도 좋은 서비스

5) Mockup Section
- 출시하고자 하는 서비스의 실제 화면과 최대한 비슷하게 화면을 구성하는 영역

6) Logo & Typography Section
- 출시하고자 하는 서비스에 필요한 로고 혹은 BI & Text Type을 정의한 영역

7) Color Style Guide Section
- 출시하고자 하는 서비스에서 사용된 대표 컬러 가이드 (*일관성있는 컬러 스타일 사용)

8) Components Section
- 출시하고자 하는 서비스의 UI디자인을 얼만큼 효율적이고 일관성있게 작업할 수 있도록 할 수 있는지 보여주는 영역

9) IA Section = Workflow = Site map
- Information Architecture
- 실제 서비스가 어떤 페이지로 구성되어있는지를 한눈에 알아볼 수 있도록 소개해주는 영역 &  실제 User가 우리의 서비스를 사용할 때 어떤 기능을 사용할 수 있도록 할 것인지 정의한 영역


10) WireFraming Section
- 위에서 설명한 해당 서비스가 구체적으로 어떤 레이아웃으로 구현되는지를 소개하는 영역
(*가급적 구체적으로 다양하게 페이지를 구현)

11) Prototype Section
- 구현하고자 하는 서비스가 가운데, 핵심적인 혹은 차별화될만한 특정 페이지 위주로 구체적인 설명을 해주는 영역

1) UX/ UI 개념 정의

UX design과 UI design은 다르다

쿠팡에 들어가서 어떠한 상품을 검색하면 왼쪽에 필터링들이 있다 (브랜드 선택, 용도 등등)
네이버 쇼핑 라이브에서는 상품을 검색하면 필터기능이 없다
 =>  조별프로젝트에 클론코딩할때 넣었어야했다 => "개선"

'UI 구현화면에 넣어줘야지'

2) UX/UI 채용할때 많이 보는 기준!
   - 학원에서 템플릿으로 디자인한사람을 거른다
   - 문제의식 없이 단순히 예쁜 형태의 디자인만 가져오는거 거른다
        => UX 에 대한 개념이 없다고 생각한다
   - 문제의식을 도출해야 한다



3) 문제의식은 어떻게 도출해야하는가?
    (1) 누구를 대상으로 할것인가?
    (2) 그 사람들에게 어떠한 방식으로 해결책을 줄것인가?
            => 이 두개가 핵심이다
                => 첫 머리부분에 나와야하고 / 최종 결과 페이지에 나와야한다
          

      
4) 어떠한 주제로 어디서 부터?
 - 평소에 내가 관심있어하는 주제로
 - 어떤 페이지부터 만들어가야 하나?

UX/UI 기획. 디자너이들이 포트폴리오 준비해서 제출할때 1개~ 3개정도의 컨텐츠(=아이템) 

(1)1개 정도 : 리디자인 => 기존에 제공 되고있는 서비스를 나만의 관점과 시선으로 
문제점을 찾아서(도출해서) 해결책을 제시하는 UX/ UI => 이걸 "케이스 스터디" 라고 부른다

(2)2개 정도 : 완전 크리에이티브한 소재 => 무에서 유를 창조해낼 컨텐츠

  ==> (1)도 빡세서 (1)이 익숙해지면 (2)를해라

시중에 제공되고 있는 서비스 가운데 어떤 걸 선택해야하나?
 => 대안 : 관심있는거 하지마라! / 시장규모가 큰 산업군
      잡코리아나 사람인 갔을때 많이 나오는 업체들의 산업군(에이전시. 자동차. 화장품. 식품) 
       => e-commerce 시장 / Health.care 시장
        => 3개 페이지정도를 봐라 => 가장 많이 채용하는 시장 
         => 시장이 작은쪽 분야를 만들면 3초이내로 안본다고 한다
